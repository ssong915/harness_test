# Reflect Plan

## Purpose

Compare meaning candidates against the user's intended meaning.

## Success Criteria

- Reads the active cycle's `### Wonder` section.
- Compares one meaning candidate at a time.
- Records user-intended nuance, model-interpreted nuance, and same/different notes.
- Writes only the active cycle's `### Reflect` section.

## Boundaries

- Do not add new meaning candidates; use `wonder`.
- Do not merge meanings; use `refine`.
- Do not produce a goal; use `restate`.
