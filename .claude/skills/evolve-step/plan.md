# Evolve-step Plan

## Purpose

Review an existing Seed for blind spots and produce a user-approved next Seed.

## Success Criteria

- Preserves Seed v1 exactly in the audit.
- Produces exactly six blind-spot candidates.
- Waits for user accept/reject decisions.
- Applies only accepted candidates to v2.
- Replaces top-level `## Seed` and appends audit history.

## Boundaries

- Do not create the first Seed; use `socrates`.
- Do not define ontology; use `ontology`.
- Do not orchestrate convergence; use `interview-harness`.
